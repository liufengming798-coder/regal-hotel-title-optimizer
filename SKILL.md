# 富豪酒店标题优化技能

## 核心目标

标题不是越短越好，而是在 20 字符限制内尽量装进更多有效信息。

合格标题要同时回答：

```text
用户是谁：游客 / 学生 / 商务 / 亲子 / 转机 / 看展看演出
去哪里：香港 + 酒店名 / 地标景点 / 酒店类型
为什么选：位置 / 交通 / 接驳 / 步行 / 配套 / 性价比
结果是什么：省心 / 少折腾 / 不赶路 / 更从容 / 更顺
```

## DeepSeek 执行规则

后续生成标题时，优先使用 `scripts/deepseek_title_generate.py` 调用 DeepSeek API 生成候选标题，再按本 skill 的质量校验复核。

调用前检查：

- 必须有环境变量 `DEEPSEEK_API_KEY`。
- 默认模型为 `deepseek-v4-pro`。
- 默认关闭 thinking：`{"thinking": {"type": "disabled"}}`。
- 不要把 API Key 写入 GitHub、文档、聊天回复或标题内容。
- 如果缺少 `DEEPSEEK_API_KEY`，先说明需要配置 key，再用本 skill 规则给出本地保守候选，不要假装已经调用 DeepSeek。

常用调用：

```bash
python scripts/deepseek_title_generate.py --count 10 "正文或 brief"
```

多行正文可通过 stdin：

```bash
cat note.txt | python scripts/deepseek_title_generate.py --count 10
```

## 硬规则

- 标题总长不超过 20 个字符。
- 优先写到 18-20 字符；低于 16 字符通常要重写，除非用户明确要极短标题。
- 一个汉字、数字、英文字母、空格或标点符号都按 1 个字符计算。
- 不使用 emoji。
- 标题默认只出现一次“香港”；不强制重复“香港”。
- 可以灵活使用“丨”“+”“，”“！”增强结构感和人感；每条优先 1-2 个标点，不要堆满。
- 不编造正文没有的酒店名、地标、距离、分钟数、权益、价格、活动或承诺。
- 数字表达优先使用正文事实；正文没有具体分钟数时，可以用“2天1夜”“3天2晚”“5天4晚”“7天6晚”等旅行行程词，但不要虚构距离或价格。
- 不使用“必住”“全网最低”“最划算”“闭眼冲”等绝对化或夸大表达。

## 生成流程

### 1. 先用 DeepSeek 生候选

把正文、brief、酒店事实和硬规则一起交给 `scripts/deepseek_title_generate.py`，要求 DeepSeek 输出 10 个候选标题。

### 2. 再拆正文四象限

```text
人群：游客 / 留学生 / 亲子家庭 / 商务出行 / 转机旅客 / 看展看演出人群
场景：几天几夜、到港第一晚、转机、逛景点、看展、亲子出行、赶飞机
买点：酒店名、位置、地标距离、交通方式、配套、早餐、房间、设施
情绪：少换乘、省心、不赶路、不折腾、行程更顺
关键词：香港、酒店名、香港酒店、地标、行程词
```

### 3. 选择信息密度最高的结构

优先使用下面结构。标题要有“小红书标题的读感”，不要写成平铺关键词。

```text
香港行程词丨酒店名+核心买点
香港地标词丨酒店名+量化交通
香港人群场景，酒店名+情绪收益
香港地标/景点+酒店名，情绪收益
香港行程词，地标/景点+酒店名+少折腾
```

### 4. 用标点做结构

标点也算字符，但好标点能提升标题观感。按需使用：

- “丨”：主副标题分层，适合“香港3天2晚丨丽豪航天城+接驳”。
- “+”：并列两个买点，适合“机场+迪士尼”“早餐+泳池”“11SKIES+亚博”。
- “，”：从场景转情绪，适合“看展住这，不怕雨”“带娃出门，少折腾”。
- “！”：只用于强情绪收口，适合“机场早班，稳了！”；不要每条都用。

标点使用原则：

- 优先让标题形成两段或三段，不要整句平铺。
- 每条标题最多出现 2 个结构标点。
- 不为了标点牺牲事实准确性；标点不能替代酒店名、地标或买点。

### 5. 压到 18-20 字符

保留优先级：

1. 一次“香港”
2. 酒店名或酒店类型
3. 地标/景点/出行场景
4. 数字化表达
5. 买点
6. 情绪收益

压缩时删除：

- “真的”“超”“太”等弱信息词
- 可有可无的标点和空格
- 重复的方便、省心类近义词
- 没有正文依据的权益和距离
- 让标题变口号但不增加信息的感叹词

## 质量校验

DeepSeek 输出后，每个候选标题都必须二次检查：

- 是否小于等于 20 字符。
- 是否优先达到 18-20 字符；低于 16 字符要说明为什么保留。
- 是否只出现一次“香港”。
- 是否无 emoji。
- 是否灵活使用 1-2 个结构标点，且观感自然。
- 是否至少包含一个酒店词或酒店类型词。
- 是否至少包含一个地标、行程词或量化词。
- 是否把买点连接到情绪收益，而不是只堆关键词。
- 是否没有虚构正文事实。
- 是否确实来自 DeepSeek 输出；如果人工改写过，要再次重新计数字符。

## 输出格式

输出 10 个标题时使用：

```text
1. 标题（N 字符）- 类型：搜索承接 / 行程攻略 / 地标交通 / 人群场景 / 情绪种草
```

最后补充：

```text
提炼依据：
人群：
场景：
买点：
情绪：
关键词：

需要确认：
只列可能影响标题真实性的事实缺口；没有就写“无”。
```

## 示例

正文信息：

```text
人群：来港游客、亲子家庭、转机旅客
场景：香港 2天1夜或3天2晚，想少换乘
买点：丽豪航天城，近亚博，步行约2分钟到11SKIES，有机场/东涌/迪士尼接驳
情绪：少折腾、更省心
关键词：香港、香港酒店、迪士尼、机场、亚博、11SKIES
```

可用标题：

```text
香港2天1夜丨丽豪航天城+机场巴士
香港迪士尼丨丽豪航天城+接驳省心
香港亚博旁，丽豪航天城步行约2分钟
香港亲子游丨丽豪航天城+泳池早餐
```

不可用标题：

```text
香港游住这省一半力
原因：有“香港”但没有酒店词、地标词或明确量化信息，信息密度不足，观感也太泛。

港玩选这家太省心
原因：没有出现“香港”，没有酒店词或地标词。

香港迪士尼免费接驳超划算
原因：“免费接驳/划算”必须有正文事实支撑，且信息密度仍可继续提升。

香港丽豪航天城机场迪士尼东涌亚博都方便
原因：信息堆叠太满，没有标点分层，读感差。
```
